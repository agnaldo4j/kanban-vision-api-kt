## Proposed cycle P6

Findings → gaps, in execution order (unblock the build first; planning and progress live on [board #6](https://github.com/users/agnaldo4j/projects/6)):

| Gap | Type | Scope | Characteristic addressed |
|---|---|---|---|
| **GAP-AN** | [E] | Build modernization: Detekt 2.x (removes the Gradle-10 deprecation and unblocks jvmTarget > 22) + configuration cache. **Java 25 daemon**: an empirical check on 2026-07-02 (CLAUDE.md, PR #205) showed Kotlin 2.4.0 compiles on a Java 25 daemon with Gradle 9.6.1 and that the observed blocker is Detekt 1.23.8's embedded compiler (tasks abort with `> 25.0.3`) — the comments in `gradle.properties`/CI still describe the old understanding (Kotlin as blocker) and must be updated in the gap. Acceptance criterion in ADR-0024: `testAll` green on a Java 25 daemon, or the claim explicitly scoped down if another blocker appears. Touches policy-immutable configs → **ADR-0024 accepted before any code** | Maintainability |
| **GAP-AO** | [M] | Supply chain: SCA (CVE scan) + SBOM in CI — tool choice (OWASP dependency-check vs CycloneDX + osv-scanner) in the design session | Security |
| **GAP-AP** | [M] | PITest on `usecases` — low initial threshold, gradual increase (same path as domain: 38% → 58%) | Maintainability |
| **GAP-AQ** | [M] | Architecture fitness functions (Konsist): hexagonal dependency rule, absence of cycles, ports in `usecases/repositories/` — automates what the audit checked by hand | Flexibility |
| **GAP-AR** | [M] | Load tests (k6 or Gatling) against docker compose + a p95 latency baseline — the first performance signal | Performance efficiency |
